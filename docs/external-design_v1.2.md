## 1. 目的・コンセプト

ブラウザ上で大阪の鉄道網を立体模型として再現するWebアプリ。
参考: TOKYO METRO 3D (https://tokyo-metro-3d-model.netlify.app/)

参考サイトとの差別化ポイント:
1. **複数事業者を収録**(Metro + JR + 主要私鉄5社)
2. **レイヤ操作**(事業者・路線単位の表示ON/OFF)
3. **駅別乗降人員の可視化・比較**

## 2. 用語定義

| 用語 | 意味 |
|---|---|
| 国土数値情報 | 国土交通省が無償公開する地理空間データ群 |
| N02 | 国土数値情報の鉄道データ(路線形状・駅位置) |
| S12 | 国土数値情報の駅別乗降客数データ |
| GTFS | 公共交通の時刻表・経路の標準データ形式。日本では主にバス向け |
| ヘッドウェイ方式 | 実時刻表ではなく「◯分間隔」の設定から列車を自動生成する近似方式 |
| 直通運転 | 異なる事業者の路線間を列車が乗り入れて走ること(例: 御堂筋線⇔北大阪急行南北線) |

### 3.2 地理境界ルール【v1.2改訂】

原則は従来どおり:
- **「大阪市内の全駅 + 市境を越えた先の1駅」**まで収録する(市境でぶつ切りにすると線が不自然に途切れるため、1駅先まで描いてフェードアウト)。
- データ構造は路線単位で独立させ、将来「京阪神広域」へ拡張可能にする。

**例外規定(v1.2で追加)**:
- Osaka Metro各線と**相互直通運転を行い、実質的に一体運行される近接路線**は、当該路線の自然な終端まで**全区間を例外的に収録できる**。
- 第1弾として**北大阪急行電鉄南北線(江坂〜箕面萱野)**を収録する。判断根拠:
  1. 御堂筋線と相互直通運転しており、運行上は「箕面萱野〜なかもず」で一体
  2. 駅番号が御堂筋線と連番のMシリーズで付番されている(模型的にも御堂筋線の北の延長として自然)
  3. 単一路線で完結し、延伸後も専用駅は5駅と小規模でスコープ影響が限定的
- **この例外は上記条件を満たす路線に限定する**。他の直通関係(堺筋線⇔阪急京都線、中央線⇔近鉄けいはんな線 等)は相手が独立した大規模ネットワークのため、例外を適用せず**従来の「市内+1駅」ルールで扱う**(=市内区間のみ)。将来の広域化で再検討。

### 3.3 対象外(将来拡張候補)【v1.2改訂】

- バス、大阪モノレール、阪堺電車、近鉄けいはんな線(長田以遠)
  - ※**北大阪急行(江坂以遠)はv1.2で対象に変更**(3.1/3.2参照)
- リアルタイム運行情報、経路検索、運賃計算
- 実時刻表ベースの正確なダイヤ再現(データが存在しないため。7章参照)
- 事業者間の直通運転の再現(R2。当面は各路線内で折返し)

## 4. 機能要件

| ID | 機能名 | 内容 |
|---|---|---|
| F1 | 3D路線網表示 | 路線を公式ラインカラーのチューブで、駅をマーカーで立体表示。地下路線は深度を持たせて地面(半透明)の下に描画。地面には簡易的な海岸線・主要河川ライン・グリッドを表示 |
| F2 | レイヤ操作 | 「事業者 > 路線」の2階層ツリーでチェックON/OFF。路線の単独表示(ソロ)ボタン。地下/地上・高架のフィルタ。非表示路線の列車・乗降人員バーも連動して消える |
| F3 | 列車運行シミュレーション | ヘッドウェイ方式。時間帯別の運転間隔を路線ごとに設定し両方向に列車を自動生成。駅で約20〜30秒停車。現在時刻連動・任意時刻ジャンプ・倍速。画面に「実ダイヤではなく運転間隔ベースの近似」と明記 |
| F4 | 駅情報表示 | 駅クリック/タップで駅名・事業者・乗り入れ路線・乗降人員(年度と出典付き)・深度(概算ラベル付き)をパネル表示 |
| F5 | 乗降人員可視化・比較 | (a)人員に比例した高さの3Dバー表示モード (b)上位ランキングパネル (c)複数ピン留めして並べて比較 |
| F6 | カメラ操作 | 軌道回転・ズーム・パン。プリセット視点(俯瞰/真上/低空)。駅クリック時のフォーカス移動 |
| F7 | クレジット表示 | データ出典と「概算・近似」の注意書きを常設表示 |
| **F8** | **ランドマーク表示【v1.2追加/R-004】** | 大阪の著名施設を**模式マーカー+名称ラベル**で地表付近に表示する。対象は5〜8箇所程度(例: 大阪城・通天閣・あべのハルカス・京セラドーム大阪・海遊館・太陽の塔 等。最終選定はG1で確定)。**位置は装飾目的の概略で精度非保証**(座標は実装時に一次情報から取得、クレジットに「装飾・概略位置」と明記)。R-002の駅名ラベル基盤(CSS2DRenderer)に相乗りし、レイヤ的にON/OFF可能とする。地理装飾(海岸線・淀川)と同じ「装飾レイヤ」に属する |

## 5. 画面設計

1画面SPA。ページ遷移なし。

```
PC想定レイアウト
+------+--------------------------------+-------+
|レイヤ |                                | 駅情報 |
|パネル |        3D ビューポート          | パネル |
|(左・畳|                                |(選択時 |
| める) |                                | のみ)  |
+------+--------------------------------+-------+
|  ◀ 時刻バー [05:00-24:00] ▶  倍速 [x60]  乗降人員バー[ON/OFF] |
+----------------------------------------------+
```

- モバイル: 3Dを全面表示。レイヤ・駅情報は下からのシート(スワイプで開閉)。時刻バーは最下部に常設
- 配色: 背景はダーク系(地下の光る路線が映えるため)
- 【v1.2】ランドマーク(F8)は装飾レイヤとしてレイヤパネル最下部に「装飾(海岸線・淀川・ランドマーク)」のON/OFFを設ける

## 7. データソース一覧

| データ | 出典 | 年度 | 信頼度ラベル |
|---|---|---|---|
| 路線形状・駅位置 | 国土数値情報 N02(実装時に最新版を再確認)。**M1は2017年系譜データで暫定運用中→T-006でN02へ差し替え(ライセンス白化)** | 2022年度〜 | [公式データ] |
| 新駅の差分 | 夢洲駅(2025年開業)・**北急の箕面船場阪大前/箕面萱野(2024年開業)** 等はN02に無い場合、一次資料の座標で手動補正 | 2024〜2025 | [公式資料+手動] |
| 乗降人員 | 各社公表の駅別乗降人員(最新年度)/ 補完: 国土数値情報 S12(実装時に再確認) | 混在 | [公式データ・年度差あり] |
| 駅の深度 | 公表資料で確認できた駅のみ実数値、それ以外は路線既定値 | - | [概算・一部推測] |
| 運転間隔 | 各社の公表時刻表から時間帯別の間隔を読み取り手動設定 | 最新 | [近似] |
| ランドマーク座標 | 各施設の公式/一次情報(実装時取得) | - | **[装飾・概略位置]** |

**データ取得経路の優先順位**(作業環境の制約でN02を直接DLできない可能性があるため):
1. GitHub上で公開されている国土数値情報の加工済みGeoJSON(実在とライセンスを確認)
2. Web経由のOpenStreetMap(ODbL表記が必要)
3. 最終手段: 一次資料ベースで座標を手動整備

## 8. 前提・制約・リスク

| ID | リスク/制約 | 影響 | 対策 |
|---|---|---|---|
| R1 | 実ダイヤのオープンデータが存在しない(GTFSはほぼバス) | 時刻表通りの運行は不可能 | ヘッドウェイ方式で近似し、UIに「近似」と明記 |
| R2 | 直通運転の存在(**御堂筋線⇔北急**、堺筋線⇔阪急、中央線⇔近鉄けいはんな、阪神なんば線⇔近鉄) | 路線単位で折り返すと実態と異なる | 各路線内で折返し、直通は将来対応。**北急も当面は江坂で折返し**。注意書きを表示 |
| R3 | 駅深度の公式データが無い | 深さ表現が概算 | 路線既定値+判明駅のみ個別値。「概算」ラベル常時表示 |
| R4 | 乗降人員の年度が事業者間で揃わない | 比較の公平性 | 駅情報パネルに年度・出典を必ず表示 |
| R5 | 描画性能 | スマホでカクつく恐れ | InstancedMesh等の最適化、レイヤOFFで負荷軽減 |
| R6 | 作業環境からN02へ直接アクセスできない可能性 | データ取得が滞る | 7章の取得経路で回避。最悪はスコープ縮小 |
| **R7** | **【v1.2】新駅・延伸のデータ鮮度**(箕面延伸2024・夢洲2025等、N02の版に含まれない駅がある) | 駅の欠落・位置ズレ | 一次資料で手動パッチ。パッチ駅は出典・取得日をコメントに残す |
| **R8** | **【v1.2】境界例外の拡大解釈**(「直通だから」と他社線まで全線収録し始めるとスコープ爆発) | スコープ管理の破綻 | 3.2の例外は「Metroと相互直通の近接小規模路線」に限定。現状は北急のみ。追加はG1承認を要する |
