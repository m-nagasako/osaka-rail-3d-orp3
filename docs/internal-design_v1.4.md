## 0. スコープ

作る: アプリの現在の表示状態をURLに載せ、そのURLを開くと同じ表示が再現される「URL共有」。

作らない(範囲外): 比較パネル強化(Nがスキップ判断)/ OGP画像(外部設計で将来対応)/ カメラ視点の共有(下記5でオプション扱い)/ N02差し替え / 直通運転。

## 1. 共有する状態

`src/core/state.js` の以下7項目を共有対象とする。

| 状態 | 型 | 既定値 | 共有 |
|---|---|---|---|
| `visibleLines` | Set<路線ID> | 全路線ON | ○ |
| `visibleStructures` | Set<'地下'/'地上'/'高架'> | 全ON | ○ |
| `simTime` | number(0時からの秒) | 28800(08:00) | ○ |
| `speed` | 1/10/60/300 | 60 | ○ |
| `running` | bool | true | ○ |
| `showRidership` | bool | false | ○ |
| `showDecorations` | bool | true | ○ |

カメラ(位置・注視点)は中央stateの外にあり、精度・容量の割に価値が低いためMVP対象外。

## 2. エンコード方式

- URLハッシュ(`#...`)を使う。静的ホスティングでサーバ設定なしに任意状態のURLがindex.htmlを返すため。
- バージョンタグ`v1`を付ける。将来フィールドを増やす場合は`v2`とし、旧`v1`リンクは壊さない。
- 既定値からの差分を基本に圧縮する。

実装スキーム:

`#v1.<L>.<S>.<P>.<F>.<T>`

- `<L>`: 路線レイヤ。ON集合とOFF集合の小さい方をID列挙し、`i`=以下ON、`x`=以下OFFで表す。件数ガード付き。
- `<S>`: 構造フィルタ。地下/地上/高架の3ビット。
- `<P>`: 倍速。`[1,10,60,300]` のインデックス。
- `<F>`: running/showRidership/showDecorationsの3ビット。
- `<T>`: `simTime`秒をbase36化。

バージョン不一致、パース不能、未知の路線ID、件数不一致、範囲外値は既定値へフォールバックし、例外を投げない。

## 3. 読み込みと書き出し

ロード時:

1. `main.js` がデータ読込後に `location.hash` を `decodeState` で解釈する。
2. `state.visibleLines` / `state.visibleStructures` / `state.simTime` / `state.speed` / `state.running` / `state.showRidership` / `state.showDecorations` へ反映する。
3. 反映後のstateを使ってworld/UIを初期化するため、初期描画とチェック状態がURLどおりになる。

共有:

1. レイヤパネル下部の「共有」ボタンを押す。
2. `encodeState` でURLハッシュを生成する。
3. `history.replaceState` で現在URLへ反映する。
4. クリップボードへコピーし、パネル内に「コピー済み」を表示する。

ライブ同期はMVP外。ボタン押下時のみURLを生成する。

## 4. テスト計画

`tests/urlstate.test.js`で以下を検証する。

- 既定、solo(1路線ON)、地下のみ、一時停止、人員バーON、装飾OFF、00:00、25:00の往復一致。
- `#`付き文字列も復元できること。
- 空文字、旧バージョン、未知ID、件数不一致、壊れた文字列が例外を投げず既定値へ戻ること。

L3では、状態を変更して共有URLを開き、レイヤ・構造フィルタ・時刻・倍速・再生/停止・人員バー・装飾が再現されることを確認する。

## 5. オプション拡張(MVP外)

- カメラ視点の共有: 要望が出た場合にv2スキームで対応する。
- OGP動的生成: 静的サイトでは難しく、外部設計どおり将来対応。

## 6. 追加UI調整

発注者Nの追加要望により、事業者単位のチェックボックスをレイヤパネルへ追加した。チェックONで当該事業者の全路線を表示、OFFで全非表示、一部路線のみ表示中は中間状態にする。これは`visibleLines`だけを更新する補助UIであり、URL共有の対象にも自然に含まれる。
