## 1. データ生成方針

`scripts/build-data.js`は引き続き「入力パッチ -> 検証 -> `public/data/*.json`生成」の一方通行とする。`public/data`は手編集しない。

M2では`LINE_DEFS`にJR西日本9路線を追加し、`operator`で事業者を区別する。大阪市外へ長く伸びる路線は`startName`/`endName`で市内+境界1駅の範囲だけを抽出する。おおさか東線は短距離で市域を出入りするため、Sprint3指示どおり全7駅を収録する。

## 2. 構造分類

駅・路線の高さ`elevM`は従来どおり「正=高架/負=地下」とする。分類は`src/sim/structure.js`に集約する。

| 条件 | 分類 |
|---|---|
| `elev < -2` | 地下 |
| `-2 <= elev <= 2` | 地上 |
| `elev > 2` | 高架 |

境界値の`2m`は`CONFIG.STRUCTURE_EPS`として定数化した。

## 3. 垂直フィルタ

状態管理は`state.visibleStructures`に`地下/地上/高架`のSetを持たせる。表示条件は既存の路線レイヤとAND合成する。

| 対象 | 制御方法 |
|---|---|
| 駅 | 所属路線のうち、路線ONかつ構造ONの組み合わせが1つ以上あれば表示 |
| 駅ラベル | 駅と同じ条件。ただしズーム距離/主要駅条件も従来どおり適用 |
| 乗降人員バー | 乗降人員ON、かつ駅の表示条件を満たす場合だけ表示 |
| 列車 | 路線ON、かつ現在位置の前後駅構造のいずれかがONの場合だけ表示 |
| 路線チューブ | 駅間セグメント単位でTubeGeometryを分割し、両端駅構造のいずれかがONなら表示 |

路線チューブは縮退実装ではなく、駅間セグメント単位の表示切替を実装済み。

## 4. JRデータの扱い

- 駅座標・駅順は`japan-train-data v0.6.0`の大阪府データを使用する。
- 乗降人員はJR西日本「データで見るJR西日本2025」の上位50駅の2024年度1日平均乗車人員を2倍し、乗降人員換算として投入する。
- 上位50に無いJR駅は注記扱いにする。
- Metro/JRの同名・同一gid駅は共有する。JR大阪とMetro梅田は別名のため別ノードとする。
- 同一ノードに複数事業者の乗降人員がある場合は概算合算値として扱い、sourceに複数事業者合算であることを残す。

## 5. 既知の近似

- JRの運行は実時刻表ではなく、時間帯別ヘッドウェイ近似。
- JRの深さ/高さは公表構造と一般的な駅構造に基づく概算。
- JR乗降人員は乗車人員×2換算であり、実測の乗降人員ではない。
- 直通運転・系統分岐は未実装。全路線とも路線内折返しで近似する。
