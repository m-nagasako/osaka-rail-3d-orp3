## 1. プロジェクト概要

OSAKA RAIL 3Dは、Osaka Metro全9路線・北大阪急行南北線・JR西日本9路線の計19路線・158駅をThree.jsで3D表示するWebアプリ。M2の範囲は、路線/駅表示、事業者別レイヤ切替、地下/地上/高架フィルタ、運転間隔ベースの列車シミュレーション、駅別乗降人員の可視化、駅クリック情報、ピン留め比較、ランキング、視点プリセット、模式的な地理装飾、ランドマーク装飾まで。

主要技術:
- Vite / Three.js / Vitest
- データ生成: `scripts/build-data.js`
- 生成データ: `public/data/lines.json`, `stations.json`, `landmarks.json`, `meta.json`
- 主要ソース: `src/core`, `src/world`, `src/ui`, `src/sim`

## 4. 品質管理方針

1. 変更前に、外部設計・内部設計・QAログ・品質管理書を確認する。
2. 変更時は、影響範囲を最小化し、`docs/qa-log.md`に理由・対処・状態を記録する。
3. データ変更後は必ず`npm run build-data`、`npm test`を実行する。
4. UI/3D変更後は`npm test`、`npm run build`に加え、品質管理書のL3チェックリスト該当項目を確認する。
5. Sprint3クローズ条件は、L1/L2合格に加えて、品質管理書v1.4のL3 26項目すべてOK。
