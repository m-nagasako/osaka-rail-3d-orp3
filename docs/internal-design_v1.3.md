## 1. データ生成方針

`scripts/build-data.js`は引き続き「入力パッチ -> 検証 -> `public/data/*.json`生成」の一方通行とする。`public/data`は手編集しない。

M3では`LINE_DEFS`に私鉄5事業者13路線を追加した。駅座標・駅順は`japan-train-data v0.6.0`の大阪府データを使用し、大阪市内+市境1駅の範囲だけを`startName`/`endName`で抽出する。

近鉄奈良線は指示書上「布施止まり」とされていたが、布施1駅だけでは路線チューブ・列車スケジュール・駅間距離検証の対象として成立しないため、Sprint4では非収録とする。布施は近鉄大阪線の境界駅として収録済み。

## 3. 乗換駅の共有/分離

基本の駅同一判定は従来どおり`gid + 駅名`で行う。梅田/東梅田/西梅田のように同じgidでも別名の駅は別ノードとして扱う。

私鉄追加では事業者をまたぐ乗換駅が別gidで入る場合があるため、指示書で共有対象とされた駅と、同名で明確に乗換となる駅だけを名前ベースで既存ノードへ統合する。

共有対象:

- 淡路、天満橋、大阪難波、大阪上本町、鶴橋、京橋、西九条、新今宮、天下茶屋、岸里玉出
- 天神橋筋六丁目、淀屋橋、北浜、九条、桜川

分離維持:

- 大阪と梅田
- 難波、なんば、大阪難波、JR難波
- 大阪阿部野橋と天王寺
- 野田と野田阪神、福島のように同名でも別駅として扱うもの

## 4. 深さ・運行・乗降人員

深さ/高さは`lines.elev[]`で路線ごとに持ち、私鉄はSprint4時点では概算値とする。阪神なんば線、京阪中之島線、近鉄難波線は地下フィルタで確認できるよう、地下駅を負値で設定した。

運行は既存ロジックと同じ時間帯別ヘッドウェイ近似で、相互直通・優等種別・区間運転は未対応。各路線内折返しとして扱う。

私鉄の駅別乗降人員は未投入。既存のMetro/JR共有駅は既存出典値を表示し、私鉄単独駅は事業者別の未投入注記を表示する。
