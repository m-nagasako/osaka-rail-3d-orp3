## 1. 目的・コンセプト

ブラウザ上で大阪の鉄道網を立体模型として再現するWebアプリ。
参考: TOKYO METRO 3D (https://tokyo-metro-3d-model.netlify.app/)

参考サイトとの差別化ポイント:
1. **複数事業者を収録**(Metro + JR + 主要私鉄5社)
2. **レイヤ操作**(事業者・路線単位の表示ON/OFF)
3. **駅別乗降人員の可視化・比較**

## 2. 用語定義

| 用語 | 意味 |
|---|---|
| 国土数値情報 | 国土交通省が無償公開する地理空間データ群 |
| N02 | 国土数値情報の鉄道データ(路線形状・駅位置) |
| S12 | 国土数値情報の駅別乗降客数データ |
| GTFS | 公共交通の時刻表・経路の標準データ形式。日本では主にバス向け |
| ヘッドウェイ方式 | 実時刻表ではなく「◯分間隔」の設定から列車を自動生成する近似方式 |
| 直通運転 | 異なる事業者の路線間を列車が乗り入れて走ること(例: 阪神なんば線⇔近鉄奈良線) |

## 3. 対象範囲

### 3.1 収録路線一覧

| 事業者 | 路線 | 備考 |
|---|---|---|
| Osaka Metro | 御堂筋線・谷町線・四つ橋線・中央線・千日前線・堺筋線・長堀鶴見緑地線・今里筋線・南港ポートタウン線 | 全9路線・全駅(夢洲含む) |
| JR西日本 | 大阪環状線・JRゆめ咲線・JR東西線・おおさか東線・JR京都線・JR神戸線・大和路線・阪和線・学研都市線 | 市内区間のみ |
| 阪急 | 神戸本線・宝塚本線・京都本線・千里線 | 市内区間のみ |
| 阪神 | 本線・阪神なんば線 | 市内区間のみ |
| 京阪 | 京阪本線・中之島線 | 市内区間のみ |
| 近鉄 | 難波線・奈良線・大阪線・南大阪線 | 市内区間のみ |
| 南海 | 南海本線・高野線 | 市内区間のみ |

### 3.2 地理境界ルール

**「大阪市内の全駅 + 市境を越えた先の1駅」**まで収録する。
- 理由: 市境でぶつ切りにすると線が不自然に途切れるため、1駅先まで描いてフェードアウトさせる
- データ構造は路線単位で独立させ、将来「京阪神広域」へ拡張可能にする

### 3.3 対象外(将来拡張候補)

- バス、大阪モノレール、北大阪急行(江坂以遠)、阪堺電車、近鉄けいはんな線(長田以遠)
- リアルタイム運行情報、経路検索、運賃計算
- 実時刻表ベースの正確なダイヤ再現(データが存在しないため。7章参照)

## 4. 機能要件

| ID | 機能名 | 内容 |
|---|---|---|
| F1 | 3D路線網表示 | 路線を公式ラインカラーのチューブで、駅をマーカーで立体表示。地下路線は深度を持たせて地面(半透明)の下に描画。地面には簡易的な海岸線・主要河川ライン・グリッドを表示 |
| F2 | レイヤ操作 | 「事業者 > 路線」の2階層ツリーでチェックON/OFF。路線の単独表示(ソロ)ボタン。地下/地上・高架のフィルタ。非表示路線の列車・乗降人員バーも連動して消える |
| F3 | 列車運行シミュレーション | ヘッドウェイ方式。時間帯別(早朝/朝ラッシュ/日中/夕ラッシュ/夜間)の運転間隔を路線ごとに設定し、両方向に列車を自動生成。駅で約20〜30秒停車。現在時刻連動・任意時刻ジャンプ・倍速(1x〜300x程度)。画面に「実ダイヤではなく運転間隔ベースの近似」と明記 |
| F4 | 駅情報表示 | 駅クリック/タップで駅名・事業者・乗り入れ路線・乗降人員(年度と出典付き)・深度(概算ラベル付き)をパネル表示 |
| F5 | 乗降人員可視化・比較 | (a)駅上に人員に比例した高さの3Dバーを表示するモード切替 (b)上位ランキングパネル (c)駅を複数ピン留めして並べて比較 |
| F6 | カメラ操作 | マウス/タッチで軌道回転・ズーム・パン。プリセット視点(俯瞰/真上/低空)。駅クリック時のフォーカス移動 |
| F7 | クレジット表示 | データ出典(国土数値情報・各社公表値)と「概算・近似」の注意書きを常設表示 |

## 5. 画面設計

1画面SPA。ページ遷移なし。

```
PC想定レイアウト
+------+--------------------------------+-------+
|レイヤ |                                | 駅情報 |
|パネル |        3D ビューポート          | パネル |
|(左・畳|                                |(選択時 |
| める) |                                | のみ)  |
+------+--------------------------------+-------+
|  ◀ 時刻バー [05:00-24:00] ▶  倍速 [x60]  乗降人員バー[ON/OFF] |
+----------------------------------------------+
```

- モバイル: 3Dを全面表示。レイヤ・駅情報は下からのシート(スワイプで開閉)。時刻バーは最下部に常設
- 配色: 背景はダーク系(参考サイト同様、地下の光る路線が映えるため)

## 6. 非機能要件

| 項目 | 目標 |
|---|---|
| 性能 | PCで60fps、スマホで30fpsを目標。列車最大200編成程度・駅約250を想定 |
| 対応環境 | Chrome / Edge / Safari 最新版(WebGL2必須) |
| ホスティング | 完全静的サイト(サーバ・DB不要)。Netlify / GitHub Pages に置ける |
| 共有性 | X(Twitter)でURLシェアされる前提。OGP設定は将来対応 |
| 言語 | 日本語UI |

## 7. データソース一覧

| データ | 出典 | 年度 | 信頼度ラベル |
|---|---|---|---|
| 路線形状・駅位置 | 国土数値情報 N02(第3.1版=2022年度基準を確認済み。実装時に最新版を再確認) | 2022年度〜 | [公式データ] |
| 新駅の差分 | 夢洲駅(2025年1月開業)等はN02に無い場合、一次資料の座標で手動補正 | 2025〜 | [公式資料+手動] |
| 乗降人員 | 第一候補: 各社公表の駅別乗降人員(最新年度) / 補完: 国土数値情報 S12(確認済み最新は令和3年度版。実装時に再確認) | 混在 | [公式データ・年度差あり] |
| 駅の深度 | 公表資料で確認できた駅のみ実数値、それ以外は路線既定値 | - | [概算・一部推測] |
| 運転間隔 | 各社の公表時刻表から時間帯別の間隔を読み取り手動設定 | 最新 | [近似] |

**データ取得経路の優先順位**(作業環境の制約により国交省サイトから直接ダウンロードできない可能性があるため):
1. GitHub上で公開されている国土数値情報の加工済みGeoJSON(実装時に実在とライセンスを確認)
2. Web経由でのOpenStreetMapデータ取得(ODbLライセンス表記が必要)
3. 最終手段: 一次資料ベースで駅座標を手動整備(この場合スコープをM1に縮小)

## 8. 前提・制約・リスク

| ID | リスク/制約 | 影響 | 対策 |
|---|---|---|---|
| R1 | 大阪の鉄道に実ダイヤのオープンデータが存在しない(調査済み: 日本のGTFSはほぼバス) | 参考サイトのような時刻表通りの運行は不可能 | ヘッドウェイ方式で近似し、UIに「近似」と明記 |
| R2 | 直通運転の存在(阪神なんば線⇔近鉄奈良線、堺筋線⇔阪急、中央線⇔けいはんな線) | 路線単位で列車を折り返すと実態と異なる | M1〜M3では「各路線内で折返し」とし、直通は将来対応。注意書きを表示 |
| R3 | 駅深度の公式データが無い | 深さ表現が概算になる | 路線既定値+判明駅のみ個別値。「概算」ラベルを常に表示 |
| R4 | 乗降人員の年度が事業者間で揃わない | 比較の公平性が下がる | 駅情報パネルに必ず年度と出典を表示 |
| R5 | 描画性能(路線約30本+列車200編成+人員バー) | スマホでカクつく恐れ | InstancedMesh等の最適化(内部設計で詳細化)、レイヤOFFで負荷軽減 |
| R6 | 作業環境から国交省サイトへ直接アクセスできない可能性 | データ取得が滞る | 7章の取得経路の優先順位で回避。最悪はスコープ縮小 |

## 9. マイルストーン

| MS | 内容 | 位置づけ |
|---|---|---|
| M1 | **Osaka Metro 9路線のみで全機能を完成させる**(3D表示・レイヤ・ヘッドウェイ運行・駅情報・乗降人員バー・時刻コントロール・カメラ) | 今回の実装フェーズの必達目標。テスト・品質管理書もM1に対して作成 |
| M2 | JR西日本の市内区間を追加 | M1のデータ構造にJRを流し込むだけで動くことを検証 |
| M3 | 私鉄5社追加・比較パネル強化・URL共有機能 | 拡張フェーズ |

方針: **「路線を横に増やす」より先に「機能を縦に貫通させる」**。全社のデータ整備を先にやると、動くものが出来る前に力尽きるリスクが高いため。
